Lucene 和 Elasticearch关系

Elaticsearch概述

简称es，是一个开源的高扩展的分布式全文搜索引擎，它可以近乎实时的存储、检索数据；本身扩展性好，通过简单的RESTful API 来隐藏Luence的复杂性，从而让全文搜索变得更简单。

ElasticSearch vs Solr 总结

  1. es基本是开箱即用，非常简单。solr安装略微复杂一点

  2. Solr利用zookeeper进行分布式管理，而ElaticSearch自身就有携带的分布式管理功能。

  3. Solr支持更多格式的数据，例如：JSON、XML、CSV、而ElasticSearch仅支持JSON格式的数据。

  4. Solr官方提供的功能很多，而ElaticSearch本身更注重于核心功能，高级功能多有地方放插件提供，例如图形化界面需要kibanna友好支撑。

  5. Solr查询快，但更新索引时慢，用户电商等查询应用多；

    • Es建立索引快，即实时性查询快，用于facebook新浪等搜索。
    • Solr是传统搜索应用的有力解决方案，但ElaticSearch更适用于新兴的实施搜索应用。

  6. Solr比较成熟，又一个更大，更成熟的用户、开发和贡献者社区，而ElasticSearch相对开发维护着较少，更新太快，学习使用成本较高。
